1. 프로젝트 개요

PaiM(Project AI Manager)은 회의록·기획 문서·메모 등의 비정형 텍스트를 LLM으로 분석해 결정(decision) / 액션(action) / 이슈(issue) / 리스크(risk) 4가지 항목을 구조화하여 저장하고, 자연어 Q&A로 조회할 수 있는 AI 기반 프로젝트 관리 도구입니다.

MVP는 Streamlit 단일 앱으로 구현하였으며, 추후 React 프론트엔드로 전환을 위해 FastAPI 백엔드 스켈레톤이 함께 유지됩니다.

4. 개발 환경 셋업

1단계 — 의존성 설치

uv sync

2단계 — MySQL 실행

# 최초 실행 시 backend/db/schema.sql 자동 적용됨
docker compose up -d

3단계 — .env 파일 설정

cp .env.example .env
# OPENAI_API_KEY 등 필요 키 입력

4단계 — Streamlit 앱 실행

uv run streamlit run frontend/app.py --server.port 8502
접속 URL: http://localhost:8502

FastAPI 백엔드 (선택)

uv run uvicorn backend.main:app --reload --port 8000
FastAPI는 향후 React 프론트엔드 전환 시 사용. 현재 MVP에서는 Streamlit이 직접 백엔드 코드를 임포트해 사용.

5. 구현된 기능

문서 업로드

  • 파일 업로드 탭: .md / .txt / .pdf 멀티파일 동시 업로드
  • 텍스트 직접 입력 탭: 붙여넣기 후 즉시 처리
  • 문서 날짜 자동 추출 (정규식, 수동 수정 가능)
  • 진행바 표시 (파일 단위)
  • 일부 청크 실패 시 경고 + 부분 저장

메모리 대시보드

  • 카테고리별 요약 메트릭 카드 (4개 컬럼)
  • 탭 필터: 전체 / 결정 / 액션 / 이슈 / 리스크
  • 항목별 카드 뷰 (topic, owner, date, source 표시)

Q&A 채팅

  • 자연어 질문 → 키워드 분류기 → DB 검색 → LLM 답변
  • 프로젝트별 대화 히스토리 유지 (멀티턴)
  • 검색 경로·출처 표시 (구조화 DB / 벡터 / 복합)
  • 디버그 expander: MySQL 항목 + ChromaDB 청크 미리보기 (실서비스 시 제거 예정)

타임라인

  • 날짜순 정렬, 날짜별 그룹화
  • 수직 HTML 타임라인 (카테고리별 컬러 도트)
  • 세그먼트 컨트롤로 카테고리 필터

프로젝트 관리 (사이드바)

  • 프로젝트 생성/선택
  • 최대 5개 표시 후 expand/collapse 토글
  • 전체 너비 버튼 + active 하이라이트 (커스텀 CSS)

6. 데이터 파이프라인 상세

추출 단계 (extractor.py)

단계설명
청크 분할3000자 단위, 200자 오버랩 (_split_chunks)
LLM 추출각 청크별 function calling → ExtractionResult Pydantic 파싱
부분 실패 처리일부 청크 실패 시 PartialExtractionError raise (성공 항목 포함)
중복 제거(category, content, 정규화 날짜, source) 4개 키 기준 dedup

저장 단계 (ingestor.py)

대상내용
MySQLmemory 테이블에 항목별 INSERT (날짜 정규화 후), 실패 시 rollback
ChromaDB원문을 500자 청크로 분할, 벡터 임베딩으로 저장

날짜 정규화 (2단계)

  • SYSTEM_PROMPT에서 YYYY-MM-DD 형식 명시 요청
  • ingestor._normalize_date(): 한국어(2026년 6월 2일), 슬래시/점 형식 폴백 변환

Q&A 검색 라우팅

경로트리거 키워드검색 방식
mysql미정, 담당, 목록, 누가, 몇 개, 리스트구조화 DB 직접 조회
chroma왜, 이유, 관련, 어때, 설명, 배경벡터 유사도 검색
both (기본)위 키워드 없음MySQL + ChromaDB 복합

8. 해결된 주요 버그

증상원인해결
Unterminated string at char 4560 대용량 문서(31KB)가 max_tokens 초과로 LLM 출력 잘림 3000자 청크 분할 + finish_reason=="length" 조기 감지
Incorrect date value: '2026년 6월 2일' LLM이 한국어 날짜 형식 반환, MySQL DATE 타입 거부 SYSTEM_PROMPT에 YYYY-MM-DD 강제 + _normalize_date() 폴백
35 validation errors — topic Field required topic: str 필수 필드를 LLM이 누락 topic: Optional[str] = None + 모든 필드에 Field(description=...)
Streamlit 사이드바에 4개 자동 탭 생성 frontend/pages/를 Streamlit이 멀티페이지로 자동 탐지 디렉토리명을 frontend/views/로 변경
라디오버튼 UX 불편 Streamlit 기본 radio 위젯이 좁고 딱딱함 커스텀 CSS 전체너비 버튼 + active 하이라이트로 교체

9. 미구현 / 잔여 작업

아래 항목들은 MVP에서 구현되지 않았거나 추후 개선이 필요한 사항입니다.

기능 개선

  • ChromaDB에서 특정 청크 삭제 기능 (현재 doc_id 단위 bulk delete만 구조적으로 가능)
  • 문서 삭제 시 ChromaDB 연동 삭제
  • 프로젝트 수정/삭제 기능
  • 메모리 항목 수동 수정/삭제 UI
  • Google Gemini LLM 구조화 추출 구현 (현재 NotImplementedError)

기술 부채

  • Q&A 검색 라우터: 키워드 규칙 기반 → LLM 기반 의도 분류로 개선 여지
  • ChromaDB 임베딩 모델 명시적 지정 (현재 ChromaDB 기본 모델)
  • MySQL 연결 풀링 미적용 (현재 매 요청마다 get_connection())
  • FastAPI 엔드포인트 구현 (현재 스켈레톤만 존재)

인프라

  • Docker Compose에 Streamlit 컨테이너 추가
  • ChromaDB 컨테이너화 (현재 로컬 파일시스템 의존)
  • 프로덕션 환경 배포 설정 (Nginx, HTTPS 등)

React 전환 (장기)

  • backend/main.py에 FastAPI 엔드포인트 구현
  • Streamlit → React 프론트엔드 마이그레이션

10. 실행 방법 요약

전체 스택 실행

# 1. MySQL 시작
docker compose up -d

# 2. Streamlit 앱 실행
uv run streamlit run frontend/app.py --server.port 8502

# 3. 브라우저 접속: http://localhost:8502

# .env에서 변경
LLM_PROVIDER=claude        # claude | openai | google

데이터 초기화 (주의)

# MySQL 데이터 전체 삭제
docker compose down -v

# ChromaDB 벡터 데이터 삭제
rm -rf .chroma/
