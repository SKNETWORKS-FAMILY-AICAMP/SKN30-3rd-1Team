## 1. 지난 액션 이행 확인

이수진(백엔드): 회원 시스템 DB 스키마 설계 완료했고, 이메일 인증 로직도 구현 중입니다. 이메일 발송은 AWS SES로 정했습니다. 비용이 저렴하고 우리 인프라가 이미 AWS 기반이라 연동이 쉽습니다.

박현우(프론트/앱): Flutter 셋업 완료했고 로그인/회원가입 화면 붙이는 중입니다.

정예린(디자이너): 관심사 카테고리 제안을 가져왔습니다. 운동, 스터디, 취미, 문화, 푸드, 반려동물 6개 대분류에 각 하위 항목을 두는 구조입니다.

---

## 2. 논의

한지민(QA·마케팅): 카테고리가 너무 많으면 초기 사용자가 선택에 피로를 느낄 수 있습니다. 6개 대분류는 적당한데, 하위 항목은 너무 세분화하지 않는 게 좋겠습니다.

정예린(디자이너): 그럼 하위 항목은 대분류당 최대 5개로 제한하겠습니다.

김태호(PM): 좋습니다. 진행 상황을 보니 회원 시스템은 순조롭네요. 이 페이스를 유지합시다.

이수진(백엔드): 한 가지 우려가 있습니다. 이메일 인증 방식으로 테스트해보니, 인증 메일이 스팸함으로 가는 경우가 종종 있습니다. 이 부분은 도메인 신뢰도 설정으로 개선하겠지만 완전히 막긴 어렵습니다.

---

## 3. 결정사항

**[결정 1]** 관심사 카테고리를 6개 대분류(운동/스터디/취미/문화/푸드/반려동물), 대분류당 하위 최대 5개로 확정한다.
이유: 초기 사용자 선택 피로를 줄이면서도 충분한 분류를 제공.

**[결정 2]** 이메일 발송 서비스를 AWS SES로 확정한다.
이유: 비용 효율적이며 기존 AWS 인프라와 연동 용이.
