## 1. 긴급 안건 — 이메일 인증 문제 심화

한지민(QA·마케팅): 회원가입 플로우 QA를 해봤는데, 이메일 인증 문제가 생각보다 심각합니다. 테스트 계정 20개 중 7개가 인증 메일이 스팸함으로 갔고, 2개는 아예 도착하지 않았습니다. 실제 사용자라면 상당수가 가입 단계에서 이탈할 겁니다.

이수진(백엔드): SPF/DKIM 설정은 마쳤는데도 이 정도입니다. 신규 도메인이라 발신 신뢰도가 쌓이는 데 시간이 걸립니다. 출시 전까지 완전히 해결되긴 어려울 것 같습니다.

김태호(PM): 이건 출시 성패가 걸린 문제네요. 가입 단계 이탈은 치명적입니다.

한지민(QA·마케팅): 3주 전에 소셜 로그인 이탈률 우려를 말씀드렸었는데, 지금 상황을 보면 오히려 소셜 로그인이 이 문제를 근본적으로 해결합니다. 소셜 로그인은 이메일 인증 과정 자체가 없으니까요.

박현우(프론트/앱): Flutter에서 카카오/구글 소셜 로그인 SDK 연동은 생각보다 어렵지 않습니다. 각 2~3일 정도면 붙일 수 있습니다.

이수진(백엔드): 저도 재검토해보니, 소셜 로그인을 넣으면 이메일 인증 서버 부담도 줄고 스팸 문제도 우회됩니다. 초기에 부담이라고 판단했던 게 지금 상황에선 오히려 반대네요.

## 2. 로그인 방식 변경 결정

김태호(PM): 상황이 바뀌었습니다. 3/9 회의에서 이메일 방식으로 정하고 소셜 로그인은 출시 후로 미뤘는데, 이메일 인증 문제가 이렇게 클 줄 몰랐습니다. 소셜 로그인을 MVP에 포함하는 걸로 결정을 변경하는 게 맞다고 봅니다.

한지민(QA·마케팅): 카카오 로그인은 국내 타깃에 필수적이고, 구글은 글로벌 대응용으로 좋습니다.

---

**[결정 1]** 로그인 방식을 이메일 + 비밀번호 단독에서 → **소셜 로그인(카카오/구글)을 MVP 핵심으로 추가**하는 것으로 변경한다. 이메일 방식은 보조 수단으로 유지한다.
이유: 3/9에는 개발 속도를 이유로 소셜 로그인을 출시 후로 미뤘으나, 이메일 인증 메일의 스팸함 유입·미도착 문제(테스트 20건 중 9건 문제 발생)로 가입 이탈 리스크가 확인됨. 소셜 로그인이 이 문제를 근본적으로 우회하므로 MVP에 포함하는 것이 타당.

> ※ 이 결정은 3/9 회의록 [결정 3]을 변경한 것임.

## 3. 모임 기능 착수

이수진(백엔드): 모임 데이터 모델과 위치 검색 API 설계는 완료했습니다. 500m 반경 검색은 PostGIS로 잘 동작합니다.

정예린(디자이너): 모임 탐색 화면은 지도 뷰와 리스트 뷰 두 가지로 제안합니다.

**[결정 2]** 모임 탐색 화면을 지도 뷰 + 리스트 뷰 토글 방식으로 구현한다.
이유: 위치 기반 앱 특성상 지도가 직관적이나, 목록 선호 사용자를 위해 리스트도 제공.

**[결정 3]** 모임 생성 시 최소 2명, 최대 20명으로 인원을 제한한다.
이유: 초근접 소규모 모임 콘셉트에 부합.
