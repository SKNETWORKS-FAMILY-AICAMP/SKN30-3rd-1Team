## 3. AI 기능 상세 논의

### 3.1 스마트 검색 아키텍처 (한상우 발표)

```
[사용자 질의] → [Query Embedding] → [Vector Search] → [Reranking] → [LLM 답변 생성]
```

- Embedding 모델: text-embedding-3-small
- Vector DB: ChromaDB (초기) → Pinecone (스케일업 시)
- Reranker: Cohere Rerank API

### 3.2 데이터 파이프라인
1. 문서 수집: 기존 문서 저장소 연동
2. 청킹: 500 tokens, 100 overlap
3. 임베딩 생성: 배치 처리
4. 인덱싱: ChromaDB 저장

### 3.3 예상 성능
- 검색 응답 시간: < 2초
- 정확도 목표: Top-5 적합률 80%
