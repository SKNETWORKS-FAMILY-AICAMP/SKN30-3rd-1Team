## 1. 코드 리뷰 세션

### 1.1 PR #287 리뷰: 사용자 인증 모듈 리팩토링

**작성자:** 홍길동

**변경 사항:**
- JWT 토큰 검증 로직 분리
- 리프레시 토큰 자동 갱신 기능 추가
- 로그아웃 시 토큰 블랙리스트 처리

**리뷰 피드백:**
- [이영희] 토큰 블랙리스트를 Redis에 저장하는 방식 좋음. TTL 설정 확인 필요.
- [박민수] `verify_token` 함수에서 예외 처리가 너무 광범위함. 구체적인 예외로 분리 권장.
- [최수진] 테스트 커버리지가 65%로 낮음. 엣지 케이스 테스트 추가 필요.

**결정:** 피드백 반영 후 재리뷰. 목요일까지 수정 완료 목표.

### 1.2 PR #291 리뷰: 결제 API 성능 최적화

**작성자:** 김지훈

**변경 사항:**
- N+1 쿼리 문제 해결 (주문 조회)
- 데이터베이스 인덱스 추가
- 캐싱 레이어 도입 (Redis)

**성능 개선 결과:**
- 응답 시간: 850ms → 120ms (86% 개선)
- DB 쿼리 수: 47개 → 3개

**리뷰 피드백:**
- [이영희] 성능 개선 효과 훌륭함. 캐시 무효화 전략 문서화 필요.
- [박민수] 인덱스 추가로 인한 쓰기 성능 영향 모니터링 필요.

**결정:** 승인. Staging 배포 후 1주일 모니터링.

## 3. 아키텍처 개선 논의

### 3.1 마이크로서비스 전환 계획

**현재 상태:** 모놀리식 아키텍처
**목표:** 핵심 도메인 마이크로서비스 분리

**분리 대상 후보:**
- 인증 서비스 (Auth Service)
- 결제 서비스 (Payment Service)
- 알림 서비스 (Notification Service)

**논의 내용:**
- [이영희] 인증 서비스부터 분리 시작 제안. 가장 독립적이고 영향 범위 작음.
- [박민수] 분리 전 API Gateway 도입 필요. Kong 또는 AWS API Gateway 검토.
- [김지훈] 서비스 간 통신은 gRPC vs REST 검토 필요.

**결정:** 다음 분기에 인증 서비스 분리 PoC 진행. 김지훈 담당.

### 3.2 CI/CD 파이프라인 개선

**현재 문제:**
- 빌드 시간 15분 이상 소요
- 테스트 환경 부족으로 대기 시간 발생

**개선 방안:**
- Docker 레이어 캐싱 최적화
- 병렬 테스트 실행 도입
- 테스트 환경 자동 프로비저닝

**결정:** DevOps 팀과 협의하여 4월 중 개선 착수.
