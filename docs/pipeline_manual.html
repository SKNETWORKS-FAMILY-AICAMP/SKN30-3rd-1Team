🔍 PaiM 평가 파이프라인

회의록을 넣으면 검색 품질을 자동으로 측정해주는 도구예요.
명령어 하나로 인제스트 → 평가 → 비교표까지 한 번에!

🤔 이게 뭐하는 도구예요?

회의록 파일을 AI 시스템에 넣고, "AI가 얼마나 잘 대답하는지"를 자동으로 측정해줘요.

예를 들어 이런 걸 알 수 있어요:

  • 임베딩 모델을 text-embedding-3-small vs text-embedding-3-large로 바꾸면 답변이 얼마나 좋아질까?
  • 청크 크기를 300자로 자르는 게 나을까, 600자로 자르는 게 나을까?
  • MySQL 검색만 쓸 때 vs 벡터 검색만 쓸 때 vs 둘 다 쓸 때 중에 어느 게 제일 좋을까?

0 시작 전 준비물

1. 파일 위치 확인

PaiM/
├── eval/
│ ├── docs/ ← 여기에 회의록 파일 넣기 (.md 또는 .txt)
│ ├── qa/
│ │ └── qa_set_A.json ← 질문-모범답변 세트
│ └── results/ ← 결과가 여기 저장됨 (자동 생성)
└── scripts/
└── run_pipeline.py ← 이 스크립트를 실행!

2. .env 파일에 아래 항목이 있는지 확인

# 필수
OPENAI_API_KEY=sk-...
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=...
DB_NAME=paiM_migration_test

# LangSmith 트레이싱 (선택, 있으면 자동 활성화)
LANGCHAIN_TRACING_V2=true
LANGCHAIN_API_KEY=ls__...

3. 실행 위치

터미널에서 PaiM 폴더 안에서 실행해야 해요.

$ pwd
/home/.../PaiM # 이 경로여야 해요

1 가장 간단한 실행

처음이라면 이것만 기억하세요!

$ uv run python scripts/run_pipeline.py \
    --docs-dir eval/docs \
    --qa-file eval/qa/qa_set_A.json \
    --project-id 12

이렇게 하면 자동으로 이걸 해줘요:

  1. 인제스트 — eval/docs/ 안의 회의록을 DB에 저장
  2. eval 3회 — sql / vector / both 모드로 각각 평가
  3. TSV 생성 — eval/results/eval_comparison.tsv 저장

2 옵션 설명

옵션 설명 기본값
--docs-dir 회의록 파일이 들어있는 폴더 경로 - 필수
--qa-file 질문-모범답변 JSON 파일 경로 - 필수
--project-id DB에 있는 프로젝트 번호 - 필수
--embed-models 임베딩 모델 이름. 여러 개 입력 가능 text-embedding-3-small 선택
--chunk-sizes 청크 크기 (문자 수). 여러 개 입력 가능 600 선택
--chunk-overlap 청크 간 겹치는 문자 수 150 선택
--search-modes 검색 방식. sql vector both 중 선택 sql vector both 선택
--workers 동시에 처리할 질문 수 (높을수록 빠름, API 제한 주의) 5 선택
--skip-ingest 이미 저장된 데이터가 있으면 인제스트 건너뜀 false 선택
--no-judge AI 점수 평가 생략 (빠르게 테스트할 때) false 선택
--output-dir 결과 파일 저장 폴더 eval/results 선택
--tag 특정 태그 질문만 평가 (예: decision action) 전체 선택
--judge-model 점수 매길 때 쓸 LLM 모델 gpt-4.1-mini 선택

3 상황별 사용 예시

💨 빠르게 결과만 보고 싶을 때 (AI 점수 없이)

$ uv run python scripts/run_pipeline.py \
    --docs-dir eval/docs \
    --qa-file eval/qa/qa_set_A.json \
    --project-id 12 \
    --no-judge \
    --workers 5

🔄 이미 인제스트된 데이터로 eval만 다시 돌릴 때

$ uv run python scripts/run_pipeline.py \
    --docs-dir eval/docs \
    --qa-file eval/qa/qa_set_A.json \
    --project-id 12 \
    --skip-ingest \
    --workers 5

🧪 임베딩 모델 2개 × 청크 크기 2개 비교 실험

$ uv run python scripts/run_pipeline.py \
    --docs-dir eval/docs \
    --qa-file eval/qa/qa_set_A.json \
    --project-id 12 \
    --embed-models text-embedding-3-small text-embedding-3-large \
    --chunk-sizes 300 600 \
    --search-modes vector both \
    --workers 5
💡 이 경우 자동으로 만들어지는 조합
2 embed_models × 2 chunk_sizes = 4번 인제스트
4 조합 × 2 search_modes = 8번 eval 자동 실행

🎯 특정 태그 질문만 테스트할 때

$ uv run python scripts/run_pipeline.py \
    --docs-dir eval/docs \
    --qa-file eval/qa/qa_set_A.json \
    --project-id 12 \
    --skip-ingest \
    --tag decision

4 결과 파일 설명

파이프라인이 끝나면 eval/results/ 폴더에 파일이 생겨요.

eval/results/
├── eval_3small_c600_sql_judged.json ← sql 모드 상세 결과
├── eval_3small_c600_vector_judged.json ← vector 모드 상세 결과
├── eval_3small_c600_both_judged.json ← both 모드 상세 결과
└── eval_comparison.tsv ← 📊 전체 비교표 (엑셀로 열기)

eval_comparison.tsv 컬럼 구성

컬럼명 설명
ID 질문 번호 (A1, B3 등)
TAG 질문 유형 (decision / action / conflict / hallu)
질문 질문 내용
예상답변 모범 답변
모범출처 모범 답변이 어느 회의록에서 온 건지
judge_[설정명] 해당 설정의 AI 평가 점수 (0~10)
sim_[설정명] 코사인 유사도 점수 (0~1, 높을수록 유사)
답변_[설정명] 해당 설정으로 AI가 실제로 한 답변
검색출처_[설정명] AI가 어떤 문서를 검색해서 답변했는지
💡 TSV 파일 여는 방법
구글 시트: 파일 → 가져오기 → 탭으로 구분된 값 선택
엑셀: 파일 열기 → 텍스트 파일 → 구분 기호: 탭

⚠️ 주의사항

임베딩 모델을 바꾸면 반드시 재인제스트!
text-embedding-3-small로 저장한 데이터를 text-embedding-3-large로 검색하면 안 돼요. 파이프라인이 자동으로 다른 컬렉션에 저장하니까 걱정 말고 돌리면 돼요.
--skip-ingest는 같은 설정일 때만!
청크 크기나 임베딩 모델을 바꿨는데 --skip-ingest를 쓰면 이전 데이터로 평가해요. 새 설정으로 처음 돌릴 땐 --skip-ingest 빼고 실행하세요.
💰 API 비용 안내
질문 30개 기준:
• 답변 생성 (gpt-4.1-mini): 약 $0.02
• judge (gpt-4.1-mini): 약 $0.03
• 임베딩 (text-embedding-3-small): 약 $0.001
전체 약 $0.05~0.10 수준이에요.
