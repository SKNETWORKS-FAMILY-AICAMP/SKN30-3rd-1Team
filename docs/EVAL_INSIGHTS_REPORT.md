## 1. 질문 유형별 검색 지표

`P/R`은 context precision/recall이다. `E2-oracle`은 기대 이력 질문에 history mode를 강제한 보조 측정이며, 생성 지표는 측정하지 않았다.

| 구성 | 결정 P/R | 액션 P/R | 변경 P/R | 비번복 대조군 P/R |
|---|---:|---:|---:|---:|
| R0-sql | .117 / .548 | .074 / .917 | .083 / .900 | .288 / 1.000 |
| R0-vec | .781 / .952 | .686 / .917 | .623 / .800 | .911 / 1.000 |
| R0-both | .082 / .968 | .076 / .958 | .093 / 1.000 | .269 / 1.000 |
| E0 | .342 / 1.000 | .228 / .958 | .422 / .900 | .485 / 1.000 |
| E1 | .352 / 1.000 | .187 / .958 | .464 / 1.000 | .465 / 1.000 |
| E2-e2e | .333 / .968 | .187 / .958 | .448 / 1.000 | .486 / 1.000 |
| E2-oracle | .329 / 1.000 | .241 / 1.000 | .406 / 1.000 | .481 / 1.000 |

전체 평균 그래프는 유형별 목적을 섞어 해석을 흐릴 수 있어 제외했다. 시각화는 [HTML 보고서](EVAL_INSIGHTS_REPORT.html)에서 확인한다.

- 검색: **precision**과 **recall**을 분리해 각각 하나의 그래프로 제시한다. 각 그래프는 결정·액션·변경·비번복 대조군 패널로 나뉘며, 패널 안에 R0-sql부터 E2-oracle까지 7개 bar가 있다.
- 생성: **faithfulness**, **response relevancy**, **citation grounding**을 각각 하나의 그래프로 제시한다. 각 그래프는 유형별 패널을 가지며, 패널 안에서 Modu/CS-Bot의 E0와 E2를 비교한다.

참조용으로, 문항당 평균 컨텍스트 수는 R0-sql `76~95행`, R0-both `81~100행`, E0/E1/E2 `18~26행`, R0-vec `5행`이다. SQL 전체·광범위 행이 precision을 희석하는 주된 원인이다.

## 2. 생성 답변 품질 (final)

값의 순서는 `faithfulness / response relevancy / citation grounding`이며, 높을수록 좋다. 생성 지표는 E0와 E2-e2e에만 존재한다.

| 코퍼스·구성 | 결정 | 액션 | 변경 | 비번복 대조군 |
|---|---:|---:|---:|---:|
| Modu E0 | .869 / .497 / 1.000 | .563 / .553 / 1.000 | .723 / .512 / 1.000 | .833 / .530 / 1.000 |
| Modu E2 | .837 / .511 / 1.000 | .681 / .554 / 1.000 | .773 / .501 / 1.000 | .592 / .476 / 1.000 |

- **Faithfulness**: 답변의 주장이 검색 컨텍스트에 근거하는 정도다.
- **Response relevancy**: 답변이 질문에 직접 답하는 정도다.
- **Citation grounding**: 답변의 `(출처: 파일명)`이 실제 검색 출처와 일치한 문항 비율이다.

## 4. 핵심 인사이트

1. **E0 하이브리드 검색은 유효하다.** R0-both 대비 전체 precision은 `.096 → .334`로 개선됐고 recall은 `.971 → .981`로 유지됐다.
2. **E1의 역할은 일반 결정·액션 recall 상승이 아니라 supersede 정합성이다.** 변경 질문은 recall `.900 → 1.000`, precision `.422 → .464`가 됐다.
3. **E2의 실제 병목은 이력 감지다.** 기대 이력 질문 5개를 모두 놓쳤으므로 E2-e2e 생성 점수는 이력 체인이 의도대로 작동한 성능을 대표하지 않는다.
4. **Oracle은 잠재력을 보여주지만 답변 품질은 아직 미측정이다.** 강제 history mode에서 변경 질문 recall은 1.0이지만, oracle에는 faithfulness·relevancy·citation·답변 원문 비교가 없다.
5. **비번복 대조군은 필수 안전장치다.** E2에서 해당 유형 faithfulness가 Modu `.833 → .592`, CS-Bot `.856 → .607`로 하락했다. 이력 맥락이 번복 서사를 과장하는지 검증해야 한다.
