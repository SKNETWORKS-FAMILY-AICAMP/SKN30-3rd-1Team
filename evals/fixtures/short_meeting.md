# 짧은 회의록

날짜: 2026-06-30
참석자: 김민수, 박지현, 이서연

## 안건

프로젝트 PaiM의 문서 업로드 흐름을 점검했다. 팀은 FastAPI를 백엔드 기본 프레임워크로 유지하고, Streamlit 화면은 데모와 관리자용 업로드 화면에만 사용하기로 했다. 사용자가 업로드한 회의록과 기획서는 MySQL의 memory 테이블에 결정, 액션, 이슈, 리스크로 저장한다.

## 결정

FastAPI 업로드 API는 파일 저장 후 백그라운드 작업으로 추출과 저장을 처리하기로 확정했다. 이 방식은 사용자가 요청 직후 doc_id를 받고 상태를 조회할 수 있어 데스크톱 클라이언트와 맞다.

## 액션

김민수는 API 문서에 업로드 상태 응답 예시를 추가한다. 박지현은 Streamlit 업로드 화면의 에러 메시지를 점검한다. 이서연은 MySQL 마이그레이션 순서를 README에 정리한다.

## 이슈

현재 PDF 텍스트 추출 결과가 빈 문자열이면 사용자가 원인을 알기 어렵다. 파일 자체가 이미지 PDF인지, pypdf가 읽지 못한 것인지 메시지가 구분되지 않는다.

## 리스크

다음 배포 전에 Windows 실행 스크립트 검증이 끝나지 않으면 팀원 데모 일정이 지연될 수 있다.
